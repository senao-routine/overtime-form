# 教員時間外勤務申請フォーム

花園中学高等学校の教員向け時間外勤務申請システム。

## 概要

教員がWebフォームから時間外勤務を申請し、管理職がスプレッドシートで承認、教員がLooker Studioで承認状況を確認できるシステム。

## システム構成

| コンポーネント | 技術 | 役割 |
|---|---|---|
| フロントエンド | Next.js / React / Tailwind CSS | 申請フォームUI |
| ホスティング | Firebase Hosting | 静的サイト配信 |
| バックエンド | Google Apps Script (GAS) | API・データ処理・メール送信 |
| データベース | Google スプレッドシート | 申請データ・マスタデータ管理 |
| ダッシュボード | Google Looker Studio | 教員向け承認状況確認 |

## 機能一覧

### 申請フォーム
- 申請種類の選択（クラブ指導・模試業務・生徒募集イベント）
- 教員名の選択（スプレッドシートのマスタデータから動的取得）
- クラブ名の選択（クラブ指導選択時のみ表示、マスタデータから動的取得）
- 活動日の選択（前月22日〜当月末日の範囲制限あり）
- 業務開始時間・終了時間の入力
- 報告事項の入力
- 教員名のブラウザキャッシュ（localStorage、次回アクセス時に自動復元）

### データ保存
- **期間シート**: 活動日に基づき自動で期間シート（例: 202604期）を作成・データ追加
- **Looker Studio統合シート**: 全期間のデータを1つのシートに蓄積
- 期間の区切り: 前月22日〜当月21日を1期間として管理

### 承認機能
- 期間シートにチェックボックス（校長・事務長・副校長・教頭の4名）
- 4名全員がチェック → 「承認済み」ステータスが自動反映
- 期間シートの承認状態がLooker Studio統合シートに自動同期

### メール通知
- 申請送信時に教員のメールアドレスへ控えメールを自動送信
- メール内容: 申請種類・活動日・クラブ名・勤務時間・報告事項
- Looker Studioダッシュボードへのリンクをメールに掲載

### Looker Studioダッシュボード
- 教員がGoogleアカウントでログインすると自分のデータのみ表示（CURRENT_USER_EMAIL フィルタ）
- 申請一覧テーブル（活動日・申請種類・クラブ名・勤務時間・承認状況）
- 期間ドロップダウンで月別フィルタ
- 合計勤務時間数のスコアカード（時間単位、小数第1位）

## API仕様

### GAS WebアプリURL

GETリクエスト:
- `?type=teachers` - 教員マスタを取得（JSON配列）
- `?type=clubs` - クラブマスタを取得（JSON配列）
- パラメータなし - ヘルスチェック

POSTリクエスト:
- 申請データをJSON形式で送信
- 期間シート・統合シートへのデータ追加、メール送信を実行

## 期間の管理ルール

- 毎月22日を境に新しい期間に切り替わる
- 例: 3月22日〜4月21日の申請 → 202604期
- 例: 4月22日〜5月21日の申請 → 202605期
- 新しい期間のシートは初回申請時に自動作成される

## 年次更新手順

1. スプレッドシートの「教員マスタ」シートで教員の追加・削除・変更
2. 必要に応じて「クラブマスタ」シートを更新
3. コードの変更やデプロイは不要

## 開発・デプロイ

### ローカル開発
```bash
npm install
npx next dev
```

### ビルド・デプロイ
```bash
npx next build
firebase deploy --only hosting
```

### GASデプロイ
1. apps-script.js の内容をGASエディタに貼り付け
2. 保存 → デプロイ → 新しいバージョンで再デプロイ
3. 発行されたURLをフロントエンドのコードに反映
